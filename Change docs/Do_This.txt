# Workout App — Change Set (for a coding agent)

**Purpose:** Improvements to the current build. Each item describes the **desired end result** and an acceptance check ("Done when"). It does *not* prescribe how to implement. "Now" lines describe the current observed behaviour only to help locate the work.

**Out of scope:** Architecture, data model, libraries, styling method. The agent chooses the how.

---

## Preserve (do not regress)
These already work and are core value — keep them intact:
- Per-exercise AI coaching rationale on Program cards (e.g. plateau/progression notes with RIR).
- The "Last: …" previous-session reference shown during logging.
- The Effort (Easy / Moderate / Hard) capture after a set.
- Distinct Home states for a training day, a rest day, and a completed day.
- The week overview with per-day focus + completion counts (e.g. STR 0/5, RUN 0/1).
- "Set X of Y" progress against the planned target during logging.

---

## P1 — Correctness (do first)

**1. Empty sessions are not workouts.**
Desired result: a session with no logged working set is never saved as a workout and never contributes to XP, streaks, or history.
Done when: no "0 min" / empty entries can appear in Recent Workouts, and they award no XP and extend no streak.

**2. Previous-session reference is correctly ordered and labelled.**
Now: the "Last:" line shows sets out of order with duplicated numbers (e.g. S2, S1, S1).
Desired result: the line lists the previous session's sets in chronological order with unique, correct set numbers and accurate weight×reps.
Done when: every set appears once, numbered in the order performed.

**3. Week progress indicator reflects real progress.**
Now: the bar under the day pills appears full while the week shows 0/20 done.
Desired result: the week progress indicator accurately represents completed vs planned work — near-empty when nothing is done, filling as sessions complete.
Done when: the bar's fill matches the "X / Y done" figure.

**4. Exercise illustrations are consistently framed.**
Now: the illustration on the Log screen is off-centre/clipped and looks broken.
Desired result: every exercise illustration appears centred and uniformly sized regardless of the source image's original dimensions, with no clipping of the figure.
Done when: cycling through all exercises shows each illustration framed identically.

**5. XP / streak reflect only real sessions.**
Desired result: XP, level progress, and streak counts are derived only from sessions that meet the "real workout" bar in item 1.
Done when: deleting or excluding empty sessions changes XP/streak accordingly, and test/junk sessions never inflate them.

---

## P2 — Clarity

**6. Progress bars distinguish filled from remaining.**
Desired result: across Home XP, level bar, and the completion modal, the completed portion uses the accent and the remaining portion uses a neutral/muted track that does not read as a second value.
Done when: no progress bar uses two saturated colours that imply two different metrics.

**7. Each colour has one consistent meaning.**
Now: the same orange is used for XP rewards, weight values, progress-bar remainders, and stepper buttons.
Desired result: a single consistent colour language — one accent for primary actions, one colour reserved for rewards/XP, neutral for value tags (weights) and progress tracks. No colour carries multiple unrelated meanings.
Done when: a user can infer meaning from colour consistently across all screens.

**8. Exercise cards use informative, consistent identifiers.**
Now: cards show cryptic 2-letter badges (CH/FT/SH/AR) that don't reliably match the exercise.
Desired result: each exercise card shows a clear, consistent visual identifier for that exercise (the same illustration used in the Log view is a natural fit), not an ambiguous abbreviation.
Done when: the card visual matches the exercise and is consistent with the active-workout screen.

**9. Status elements don't masquerade as buttons.**
Now: "Today complete" is shaped like a button but does nothing.
Desired result: a non-actionable status reads as a status indicator; if it is meant to open the completed session, tapping it does that.
Done when: every button-shaped element is tappable and does something.

**10. Logging a set is the clear primary action on the Log screen.**
Now: "LOG SET" and "Next" are both prominent and similar.
Desired result: within an exercise, logging a set is visually primary; advancing to the next exercise is visually secondary until the target sets are met, so it isn't mistapped.
Done when: it's obvious at a glance which action records the current set.

**11. Recent Workouts summarises real content and opens the session.**
Desired result: each entry summarises what was done (focus + exercise count + set count), not duration alone; tapping it opens that logged session. Zero-content sessions never appear (see item 1).
Done when: entries are informative at a glance and tappable.

**12. Week-view day states are visually distinct.**
Desired result: today, the selected day, completed days, and rest days are each visually distinguishable and understandable without a legend or guesswork.
Done when: a user can tell which day is today, which is selected, and which are done, at a glance.

---

## P3 — Polish

**13. No false precision in numbers.**
Desired result: time estimates are rounded (e.g. "~19 min", not "~19m 15s"); weights show decimals only when non-whole ("80 kg", "22.5 kg", not "80.0 kg").
Done when: no "~" estimate is shown to the second, and whole-number weights have no trailing ".0".

**16. Logging beyond the planned set count is graceful.**
Desired result: when the user logs more sets than planned, the counter continues sensibly (e.g. "Set 6") rather than erroring or capping, consistent with "sets count until Next exercise."
Done when: a 6th set on a 5-set target logs cleanly.

**17. Completion modal ties reward to real work.**
Desired result: the Workout Complete screen shows a short summary of what was actually achieved (sets done, total volume, any PR) alongside the XP/streak.
Done when: the modal reflects the session's real output, not only XP.
